# 小手机信息架构与对象模型

## 1. 产品结构总览

小手机采用“手机壳 + 内容 App + 抽屉面板”的结构。

### 一级层级

1. 锁屏层
2. 桌面层
3. App 主内容层
4. 功能面板层

这四层共同形成“像手机而不是像单页网页”的体验。

## 2. 界面层级

### 2.1 锁屏

职责：

- 呈现时间与通知感
- 让角色通过“新通知”进入用户视野
- 提供进入桌面的过渡体验

### 2.2 桌面

职责：

- 承载主要模块入口
- 提供分页桌面和 dock
- 维持手机产品的主导航形式

当前桌面上的主要入口包括：

- 随聊
- 论坛
- 手账
- 相机占位
- 世界书
- 记忆
- 设置
- 导入

### 2.3 主视图

当前主视图以 tab 组织：

- 消息
- 朋友圈
- 联系人
- 我的
- 聊天页作为特殊主视图

### 2.4 功能面板

以底部抽屉形式承载次级管理功能：

- 世界书
- 记忆
- 手账
- 设置
- 导入
- Persona
- Character
- 论坛

## 3. 模块职责

### 消息

职责：

- 展示角色会话列表
- 展示摘要、未读数和最后时间
- 作为进入具体聊天的主入口

### 联系人

职责：

- 管理角色列表
- 展示角色摘要信息
- 进入角色详情与角色编辑

### 聊天页

职责：

- 展示当前会话消息
- 接收用户输入
- 触发模型调用
- 展示继续生成、查看 prompt 等操作

### 世界书

职责：

- 维护共享设定词条
- 通过关键词驱动上下文注入
- 提供角色共通背景

### 记忆

职责：

- 维护长期记忆
- 保存偏好、关系节点、剧情事件等可复用信息
- 在后续聊天中参与上下文拼装

### 朋友圈 / 论坛 / 手账

职责：

- 让聊天之外的内容有落点
- 扩展角色存在感和世界厚度
- 增强“生活空间”而不是“聊天页面”的产品感

### 设置

职责：

- 管理主题
- 管理模型接口
- 管理全局系统提示词
- 暴露当前 prompt 预览

### 导入

职责：

- 导入角色卡
- 导入世界书
- 导入聊天记录
- 缩短用户迁移与内容构建成本

## 4. 核心对象模型

### 4.1 Persona

定义：

- 用户在产品中的人设表示

核心字段：

- `name`
- `signature`
- `bio`
- `masks`

用途：

- 影响聊天中的“用户身份”
- 显示在个人页与联系人页
- 进入 prompt 拼装

### 4.2 Character

定义：

- 角色卡对象，是产品的核心内容单位

当前关键字段：

- `name`
- `subtitle`
- `description`
- `personality`
- `scenario`
- `systemPrompt`
- `summary`
- `messages`

用途：

- 决定角色表达风格
- 决定联系人与聊天页的展示
- 构成 prompt 的角色层信息

### 4.3 Chat Session

定义：

- 某个角色或群聊与用户的会话容器

当前表现：

- 消息列表
- 最后摘要
- 最后时间
- 未读数

用途：

- 承载连续互动
- 提供最近上下文
- 作为记忆和世界书命中的主要触发场域

### 4.4 Memory

定义：

- 长期记忆条目，用于稳定角色对用户与剧情的理解

当前条目类型示例：

- 偏好
- 关系节点
- 剧情事件

用途：

- 进入 prompt 拼装
- 避免重要设定只能存在于聊天历史里

### 4.5 Worldbook

定义：

- 共享设定词条集合，参考 `SillyTavern` 的 lorebook 思路

当前关键字段：

- `title`
- `text`
- `tags`
- `keys`
- `enabled`

用途：

- 在关键词命中时注入上下文
- 维护地点、规则、角色细节等共享设定

### 4.6 Feed Content

定义：

- 围绕角色生活空间的非聊天内容流

包含：

- `moments`
- `forumPosts`
- `journals`

用途：

- 扩展内容空间
- 让角色和关系推进不局限于聊天页

### 4.7 Model Settings

定义：

- 纯前端模型连接配置

当前关键字段：

- `apiName`
- `apiUrl`
- `apiKey`
- `modelName`
- `temperature`
- `maxTokens`
- `systemPrompt`

用途：

- 决定如何发起真实对话
- 决定全局系统上下文和模型行为

## 5. 当前上下文拼装结构

当前前端上下文按以下顺序组织：

1. 全局 system prompt
2. 用户 Persona
3. 当前角色卡
4. 最近长期记忆
5. 命中的世界书词条
6. 最近聊天消息
7. 当前用户输入

这是当前小手机对 `SillyTavern` 思路的核心吸收点。

## 6. 当前原型与 v1 的关系

当前原型已经覆盖：

- 主要界面框架
- 核心内容对象
- 本地状态持久化
- prompt 预览
- 模型直连

`v1` 还需要补强的方向：

- 更稳定的导入兼容性
- 更完整的角色字段和世界书编辑体验
- 更明确的多会话管理
- 更清晰的内容层级与状态反馈
