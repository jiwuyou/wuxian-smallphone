# 小手机核心流程

## 1. 首次进入流程

### 流程

1. 用户进入产品，首先看到锁屏
2. 锁屏展示时间与角色通知
3. 用户解锁进入桌面
4. 用户从桌面进入消息或联系人
5. 用户打开某个角色会话开始聊天

### 目标

- 先建立“手机感”
- 再建立“角色存在感”
- 最后进入“对话”

## 2. 聊天主流程

### 流程

1. 用户进入某个角色聊天页
2. 前端读取当前角色卡、Persona、记忆、世界书与消息历史
3. 系统拼装 prompt
4. 用户发送消息
5. 如果已配置 OpenAI 兼容接口，则发起真实请求
6. 如果未配置接口，则返回本地回退回复
7. 回复写回聊天记录，并更新摘要与时间

### 关键点

- 聊天不是孤立的文本往返，而是由角色卡和内容系统共同驱动
- 回退回复存在的意义是让纯前端原型在无接口时仍然可体验

## 3. 角色编辑影响聊天流程

### 流程

1. 用户进入联系人或角色编辑面板
2. 修改角色简介、性格、场景、补充提示词
3. 保存角色信息
4. 返回聊天页
5. 后续消息发送时，新角色信息自动进入 prompt

### 目标

- 让角色卡编辑结果即时反馈到聊天
- 让产品具备明确的“调教角色”能力

## 4. 世界书注入流程

### 流程

1. 用户在世界书中维护词条与关键词
2. 用户聊天时，系统读取最近消息和当前输入
3. 命中关键词的世界书词条进入上下文
4. 模型回复自然引用设定、地点或气氛信息

### 典型应用

- 地点设定：海街、码头、夜市
- 氛围规则：慢节奏、留白、日常碎语
- 角色细节：摄影习惯、互动偏好

## 5. 长期记忆流程

### 流程

1. 用户手动添加记忆，或在聊天中形成记忆片段
2. 系统将记忆写入本地存储
3. 后续聊天时，最近记忆优先进入 prompt
4. 模型在回复中保持关系与偏好连续性

### 当前状态

当前以“最近若干条记忆直接进入上下文”为主，尚未实现更复杂的分层记忆筛选。

## 6. 导入流程

### 支持对象

- 角色卡
- 世界书
- 聊天记录

### 流程

1. 用户选择文件或粘贴 JSON
2. 系统自动识别类型或按用户指定类型导入
3. 数据映射到本地对象模型
4. 角色、世界书或聊天记录立即可见

### 目标

- 降低从 `SillyTavern` 等工具迁移的门槛
- 提升内容起盘效率

## 7. 设置与模型连接流程

### 流程

1. 用户进入设置
2. 填写 API 名称、Base URL、API Key、模型名和采样参数
3. 设置保存到本地
4. 后续聊天时直接用该配置调用 OpenAI 兼容接口
5. 用户可在设置页查看当前 prompt 预览

### 目标

- 让纯前端版本具备真实可用性
- 让用户能理解“角色回复为什么会这样”

## 8. v0 到 v1 的过渡流程

### 当前 v0

- 手机壳和主要内容模块已建立
- 内容主要是示例数据
- 没有正式后端与 agent runtime

### v1 演进重点

1. 稳定前端角色聊天闭环
2. 稳定导入体验
3. 强化角色、世界书、记忆对回复的影响
4. 明确产品内容结构与用户心智

### v2 预留

- 工具调用
- agent 化行为
- 多层记忆与上下文压缩
- 后端 session 和跨端同步
