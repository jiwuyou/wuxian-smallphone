# 小手机产品总览

## 1. 产品定义

小手机是一个以“角色陪伴”为中心的手机形态产品。用户不是打开一个普通聊天框，而是进入一台装着角色、世界设定、长期记忆和日常内容流的小型私人手机。

产品的核心不是“问答”，而是“持续相处”：

- 角色有独立身份和说话方式
- 对话会受人设、世界书、记忆影响
- 内容不只存在于聊天页，还分布在朋友圈、论坛、手账、联系人等手机界面里
- 用户感受到的是一个可长期停留的生活空间，而不是一个单一模型入口

## 2. 不是什么

小手机不是以下产品：

- 不是通用 AI 助手工作台
- 不是开发者 agent 控制台
- 不是只有聊天页的角色扮演网页
- 不是 `SillyTavern` 的简单换壳

`SillyTavern` 提供的是角色卡、世界书、聊天组织的参考；小手机要在此基础上形成更强的手机叙事感与陪伴产品体验。

## 3. 产品定位

### 核心定位

一款“陪伴式角色聊天手机”。

### 价值主张

- 比普通聊天更沉浸：用户进入的是角色生活场域
- 比传统角色卡产品更日常：角色存在于消息、动态、记忆和世界书的统一空间里
- 比纯工具型 agent 更柔和：产品重心是关系、氛围和连续互动

### 产品气质

- 温柔、日常、留白、慢节奏
- 手机感强于网页感
- 陪伴感强于功能感

## 4. 目标用户

### 核心用户

- 喜欢角色陪伴、角色聊天和长期关系推进的用户
- 对“手机式沉浸界面”有偏好的用户
- 希望维护角色设定、世界观和长期记忆的高频聊天用户

### 次级用户

- 有 `SillyTavern` 使用经验、但希望体验更产品化界面的用户
- 想要基于角色卡做更完整内容产品的创作者

## 5. 典型场景

- 用户在锁屏看到角色通知，进入桌面，再进入消息继续聊天
- 用户修改角色卡后，角色回复语气立刻发生变化
- 用户补充世界书词条，让后续聊天自然提到地点、气氛和共通背景
- 用户把聊天碎片写入长期记忆，之后在关系推进时被重新引用
- 用户在朋友圈、论坛和手账里扩展聊天以外的内容空间

## 6. 三个参考来源

### 6.1 当前原型：`generic-mini-phone`

作用：

- 提供现有手机壳、桌面、锁屏、消息和面板式交互原型
- 提供当前纯前端的数据结构参考
- 提供 v0 到 v1 的实际起点

### 6.2 能力参考：`SillyTavern`

主要借鉴：

- 角色卡字段组织
- 世界书 / lorebook 注入方式
- 长期记忆与聊天上下文关系
- 陪伴聊天产品范式

### 6.3 未来运行时参考：`opencode`

主要借鉴：

- session 驱动模型
- agent / subagent 分层
- tool orchestration
- 长会话处理与上下文压缩机制

## 7. 当前产品状态

当前仓库里的小手机已经具备这些能力：

- 锁屏、桌面、消息、联系人、朋友圈、论坛、手账等手机 UI
- 角色基础信息维护
- 记忆、世界书、本地导入与本地存储
- 纯前端 prompt 拼装
- OpenAI 兼容接口直连

当前仍缺少：

- 正式后端
- 真实多会话与会话同步体系
- 工具调用
- agent runtime
- 面向正式上线的安全策略

## 8. v1 目标定义

`v1` 的目标不是完成 agent 化，而是完成“纯前端可用产品”：

- 用户可以维护角色卡、人设、世界书和记忆
- 用户可以导入角色卡 / 世界书 / 聊天记录
- 用户可以通过 OpenAI 兼容接口进行真实聊天
- 聊天能显式受角色卡、记忆和世界书驱动
- 产品整体维持手机形态和陪伴氛围

## 9. 成功标准

产品文档层面的成功标准：

- 新读者能快速理解小手机是什么产品
- 研发能据此理解对象模型与页面结构
- 团队能区分当前原型、v1 可用产品、v2 agent 化产品
- 后续设计、前端和后端方案能沿用同一套产品语言
