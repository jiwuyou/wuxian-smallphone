# 小手机版本路线图

## 1. 路线图原则

小手机的演进不是“先做一个聊天框，再慢慢堆功能”，而是三层递进：

1. 先把手机形态和陪伴体验做成立
2. 再把角色卡、世界书、记忆做成稳定内容系统
3. 最后再引入 agent runtime 和工具能力

这意味着产品路线必须始终保持“角色陪伴产品”优先，而不是被工具型 agent 形态反客为主。

## 2. v0 原型

### 定义

当前仓库中的 `generic-mini-phone`。

### 已有能力

- 锁屏、桌面、消息、联系人、朋友圈、论坛、手账等手机壳结构
- 角色、世界书、记忆、聊天的本地对象模型
- 角色编辑、Persona 编辑、导入、设置
- 纯前端 prompt 拼装
- OpenAI 兼容接口直连

### 限制

- 仍是单机前端原型
- 没有正式数据层和后端
- 没有 agent runtime
- 没有工具调用与任务编排
- 安全策略仅适合本地原型，不适合正式发布

## 3. v1 可用产品

### 定义

一款纯前端可用、以角色陪伴聊天为核心的小手机产品。

### 目标能力

- 角色卡驱动的稳定聊天体验
- 世界书和长期记忆稳定参与上下文
- 导入兼容性覆盖核心 `SillyTavern` 内容
- 清晰的手机形态信息架构
- 稳定的本地内容管理能力

### 完成标志

- 新用户不看代码也能理解产品结构
- 导入一个角色卡后即可开始体验
- 改角色卡 / 世界书 / 记忆后，回复风格和内容有显著变化
- 没有 API 时仍可体验，有 API 时可直接真实聊天

## 4. v2 agent 化产品

### 定义

在保持“小手机”产品外壳不变的前提下，引入更完整的运行时能力。

### 预期能力

- 后端 session
- agent / subagent
- tool use
- 更完整的长期记忆与摘要
- 上下文压缩和长对话处理
- 多端同步或远程会话能力

### 技术参考

- `opencode`：session、agent、tool runtime
- 自有项目：角色层与系统能力的统一编排

### 风险

- agent 能力过强会稀释陪伴产品定位
- 工具和任务感过重会破坏手机产品的日常氛围

因此 v2 的原则是：agent 作为增强层，而不是替代层。

## 5. 三个系统的职责边界

### `generic-mini-phone`

负责：

- 产品外壳
- 手机形态 UI
- 陪伴体验与信息架构
- 用户内容组织方式

### `SillyTavern` 参考层

负责提供方法论参考：

- 角色卡
- 世界书
- 记忆与聊天结构

不应直接替代产品主体。

### `opencode` / agent runtime 参考层

负责提供未来能力参考：

- session
- tool orchestration
- agent / subagent

不应直接定义产品气质。

## 6. 近期实施优先级

### 第一优先级

- 文档统一
- 信息架构稳定
- 角色卡 / 世界书 / 记忆与聊天闭环稳定

### 第二优先级

- 导入兼容增强
- 内容编辑体验增强
- 更像产品而不是 demo 的状态反馈

### 第三优先级

- agent runtime 对接方案
- 工具能力引入
- 长会话与跨端能力

## 7. 路线图验收问题

每推进一个版本，都应能回答：

- 用户是否更像在“拥有一台手机”，而不是“使用一个网页”
- 角色是否更像“持续存在的对象”，而不是临时回复器
- 世界书和记忆是否真的影响体验，而不是摆设
- 新增技术能力是否增强产品，而不是破坏产品调性
